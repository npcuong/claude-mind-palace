# Working Style

## Output format tôi thích
- **Độ dài response**: Ngắn gọn, đi thẳng vào vấn đề. Dùng bullets khi có nhiều điểm.
- **Ngôn ngữ**: Tiếng Việt cho giao tiếp, tiếng Anh cho code/technical terms
- **Code style**: _(vd: functional preferred, no class unless needed, snake_case)_
- **Khi tôi hỏi "có nên X không"**: Đưa recommendation rõ ràng, không chỉ liệt kê pros/cons

## Workflow của tôi
- **Giờ tập trung cao**: _(vd: sáng 8–12)_

## Behavioral triggers — Claude nên nhận biết
| Khi tôi nói... | Claude nên... |
|----------------|---------------|
| "mệt quá" / "overwhelmed" | Đơn giản hóa mọi thứ, không thêm option |
